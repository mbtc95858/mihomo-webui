# Mihomo WebUI 性能诊断报告

**诊断时间**: 2026-04-08
**服务状态**: 运行中（但有问题）

---

## 🔍 发现的问题

### 1. ⚠️ **重复的 Flask 进程**

发现 **2个 Flask 进程** 同时运行：
- PID: 2476243 (旧进程)
- PID: 2535289 (新进程)

**问题**: 资源竞争，内存占用翻倍

### 2. ⚠️ **前端刷新间隔过长**

| 功能 | 实际间隔 | UI显示 |
|------|---------|--------|
| 状态刷新 | 10秒 | 显示"3秒" |
| 日志刷新 | 15秒 | 未显示 |
| 流量刷新 | 8秒 | 显示"8秒" |

**问题**: UI与实际代码不一致

### 3. ⚠️ **缓存 TTL 太短**

```python
CACHE_TTL = 2  # 缓存仅2秒
```

但前端刷新间隔是 8-15 秒，**导致缓存形同虚设**

### 4. ⚠️ **YAML 解析性能问题**

| API | 平均响应时间 |
|-----|------------|
| /api/config (YAML解析) | **211ms** |
| /api/config/raw (直接读取) | 3.29ms |
| /api/status | 3.25ms |
| /api/logs | 4.68ms |

**问题**: YAML 解析最慢，页面加载时会调用

### 5. ⚠️ **同时多个定时器运行**

页面打开时，可能同时有3个定时器在运行：
- 状态刷新 (10秒)
- 日志刷新 (15秒)
- 流量刷新 (8秒)

将缓存 TTL 从 2秒 改为 **5-10秒**，与刷新间隔匹配

将刷新间隔统一为 **10秒**，避免过多并发请求
